General Description
FIT-NESS is a mobile fitness application developed with Flutter and Dart. The
application allows the user to select predefined workout routines, create custom
routines from an online exercise library, start workout sessions, track workout
history, estimate burned calories, and review progress through statistics and
charts.

Main Requirements Covered
1. Flutter + Dart multiplatform development:
   The project is a Flutter application with Android, iOS, Web, Windows, Linux
   and macOS folders generated by Flutter. The application code is mainly inside
   the lib folder.

2. REST API / JSON data:
   The exercise library is loaded from a remote JSON endpoint using the http
   package. The service is implemented in lib/services/exercise_api_service.dart.

3. Local database with SQLite:
   Persistent local storage is implemented with sqflite in
   lib/services/database_service.dart. The database stores completed workouts,
   cached exercises, custom routines, and the selected routine.

4. Offline-first behaviour and synchronization:
   When internet is available, exercises are downloaded from the API and cached
   locally in SQLite. When internet is not available, the app first tries to use
   the cached exercises. If the cache is empty, it uses a small fallback exercise
   list so the app remains usable. The Sync Data action in the profile menu
   refreshes the exercise data when a connection is available.

Internet / Wi-Fi Usage
- The app uses internet when opening the Exercise Library for the first time or
  when the user refreshes/synchronizes exercise data.
- The app uses internet when opening Google Maps to search for nearby gyms.
- The app does not require internet to view saved workout history, progress,
  selected routines, custom routines already created, or previously cached
  exercises.
- If there is no internet connection, the API service falls back to local cache
  and then to a built-in offline exercise list.

Google Maps Integration
The Home screen includes a "Gyms Near You" card. When the MAP button is tapped,
the app opens Google Maps with a search for gyms near the user. This is handled
through url_launcher and requires internet/GPS permissions handled by Google
Maps or the browser.

Workout Routines
- Predefined routines: Push, Pull, and Legs.
- Users can select one active routine.
- Users can create custom routines from exercises loaded from the API or cache.
- The selected routine is stored locally and shown in the Today Plan section.

Workout Sessions
When a routine is started, the app opens a workout session screen with:
- current exercise
- current set
- total sets
- timer
- progress bar
- complete set / next exercise flow

When the session finishes, the workout is stored in SQLite as a completed
workout. Each completed routine is estimated as 100 kcal burned.

Progress and History
- History shows completed workouts, date, duration, and estimated calories.
- Progress calculates total workouts, total minutes, estimated calories, weekly
  activity, and recent workouts from SQLite data.

Animations Included
The project uses flutter_animate to improve the user experience:
- Water intake animation with water drops.
- Calories card with animated fire effect.
- Workout session animations when a set is completed.
- Dumbbell/check animation during workout progress.
- Fire animation when a workout is completed.
- Animated routine cards and active routine indicator.
- Exercise library cards entering with staggered fade/slide animations.
- Progress cards, chart and recent workouts entering with smooth animations.

General File Structure
- lib/main.dart:
  App entry point, theme, and GoRouter route configuration.

- lib/models:
  Data models such as Routine, Exercise, CompletedWorkout, ProgressRecord and
  WorkoutSession.

- lib/screens:
  Main UI screens: Home, Routines, Exercise Library, Workout Details, Workout
  Session, Progress, and History.

- lib/services:
  API service, SQLite database service, and synchronization manager.

- lib/controllers:
  Progress-related logic and calculations.

- lib/widgets:
  Reusable UI components such as routine cards and bottom navigation.

- lib/data:
  Predefined local data, including the Push, Pull and Legs routines.

Main Packages Used
- go_router: navigation between screens.
- http: REST API / JSON data consumption.
- sqflite: local persistent database.
- path: database path handling.
- fl_chart: progress charts.
- flutter_animate: UI animations.
- url_launcher: opening Google Maps / external URLs.
- google_fonts: typography styling.
